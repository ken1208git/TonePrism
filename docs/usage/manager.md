# 管理ソフト（Manager）の使い方

管理ソフト（Manager）は、展示するゲームの情報や設定を管理するスタッフ用の道具です。
ウィンドウの上にあるタブで、やることが分かれています。

| タブ | できること |
| --- | --- |
| ゲーム | 展示するゲームの追加・編集・削除・バージョンの管理 |
| ストア | 来場者の画面に表示するゲームと、その並び順を決める |
| バックアップ | データの控えを取る・元に戻す |
| ログ | 動作の記録を見る（不具合の原因さがしに使う） |
| アップデート | 新しいバージョンの確認と更新 |
| 設定 | 記録の保存先やバックアップの設定など |

## ゲーム

展示するゲームを管理します。一覧の上のボタンで、追加・編集・バージョンアップ・削除ができます。

- **追加**: 「ゲーム追加」からゲームの入ったフォルダを選ぶと、TonePrism の中に取り込まれます。
- **編集**: 一覧からゲームを選んで、情報を直します。
- **バージョンアップ**: 同じゲームの新しい版を登録します。
- **削除**: 確認画面で確かめてから削除します（一覧からも、ゲームのファイルも消えます）。

!!! tip "くわしい手順はこちら"
    項目の入力や自動検出など、ステップごとの詳しい手順は
    **[ゲームの追加・編集・更新（くわしく）](games.md)** を見てください。

## ストア

来場者の「ストア」画面に出す**セクション**（ゲームが並ぶ列・棚）を作って管理します。
セクションごとに、出すゲームと並び順、見た目の形を決められます。

!!! tip "くわしい手順はこちら"
    セクションの作り方や項目の意味は **[ストアの設定](store.md)** を見てください。

## バックアップ

ゲームの登録データの**控え（バックアップ）**を取ったり、元に戻したりします。

- **手動で取る**: 「今すぐバックアップ」で、その時点のデータの控えを保存します。
- **自動で取る**: 一定の時間ごとに、管理ソフトを開いたタイミングで自動で控えを取ります（間隔は設定で変えられます）。
- **履歴**: いつ・どのパソコンで取ったか、サイズなどが一覧で見られます。
- **元に戻す（復元）**: 履歴から選んで「復元」。**戻す前に、来場者向けの画面（Launcher）をすべて閉じてください**という注意が出ます。
  実行すると今のデータは控えに退避され、選んだ控えに入れかわります。
- **ゲーム本体・案内画像も自動で控える**: バックアップを取ると、登録データ（DB）に加えて**ゲーム本体の `games` フォルダと初回説明の `guide` 画像も同時に控え**を取ります。**中身が同じファイルは 1 個にまとめて保存**するので（別ゲームが同じランタイムを積んでいても、版違いフォルダが重複していても 1 個）、**ディスクは見た目どおり小さく済みます**（多くの世代を持っても、増えるのは新しく加わったファイルの分だけ）。「最終アセット控え」に最後に取った日時と**控え全体の実使用量**が出ます。設定タブで取得の ON/OFF と世代数を変えられます。

!!! note "控えの保存場所"
    控えは保存先フォルダの中に **`asset_pool`（ファイルの実体を中身ごとに 1 個だけ置く場所）** と **`asset_snapshots`（各回の控えの「目録」）** という形で入ります。エクスプローラーで見えるサイズも、サーバーの容量計算も、**実際の使用量（重複を除いた分）どおり**に出ます（ハードリンクのように「見かけだけ大きい」という現象は起きません）。

!!! note "最初の1回だけ時間がかかります"
    ゲーム本体（`games`）・案内画像（`guide`）の控えを**取り始めた最初の1回**は、全ファイルを読み込むため時間がかかり、共有サーバーへの通信も増えます（2回目以降は変わった分だけ読むので速くなります）。アップデート直後は、混雑しない時間帯に「今すぐバックアップ」を一度押しておくと安心です。

!!! warning "ゲーム本体の「復元（元に戻す）」は今後の対応です"
    現在の「復元」は登録データ（toneprism.db）だけを戻します。**ゲーム本体（`games`）・案内画像（`guide`）の控えは取れていますが、ワンクリックで戻す機能は準備中**です。
    そのため、いまの `games` と違う時点の DB を復元すると、両者がズレて「登録はあるのに起動できないゲーム」が出ることがあります（その場合の案内は復元後のレポートに出ます）。控えからの復元は管理ソフトの機能として今後追加します（控えはファイル名がハッシュのため、手作業での復元は想定していません）。

    復元のあと、管理ソフトが自動で**整合性チェック**を行い、ズレがあれば内容（起動できないゲーム／ディスクに無いバージョン／登録に無い余分なフォルダ）と**直し方の手順**を表示します。起動できないゲームが出た場合は、その案内に従って同じ時点の `games` フォルダを補うか、退避された控え（`safety_*.db`）から戻してください。

    なお「データベースのリセット」は登録データと `games` フォルダの両方をまっさらにするため、復元のようなズレは起きません。

## ログ

来場者向けの画面（Launcher）と管理ソフト（Manager）の**動作の記録**を見られます。
どちらの記録か、また重要度（エラー／注意／情報）で絞り込めます。不具合の原因さがしに使います。

## アップデートの適用

新しいバージョンの確認と更新をします。今のバージョンが表示され、新しい版があると知らせてくれます。

「今すぐアップデート」を押すと、こう進みます。

1. 管理ソフトが**いったん閉じて新しい版で開き直す**ことを、先にお知らせします（来場者向けの画面が開いていたら閉じるよう案内が出ます）
2. 新しい版を自動でダウンロード（進み具合のバーが出ます）
3. 中身を入れかえ
4. 管理ソフトがひとりでに開き直して、**「✓ アップデート完了」**のお知らせが出ます

うまくいかなかったときは、内容が表示され、「もう一度」やダウンロードページを開く案内が出ます。

!!! warning "更新中の注意"
    更新の間、管理ソフトはいったん閉じます（先のお知らせのとおり）。
    ゲームのデータは守られるので、そのまま開き直すのを待ってください。

## 設定

記録の保存先やバックアップの設定などを調整します。

- 各まとまりは **「適用」で確定／「元に戻す」で取り消し**します（変えてもすぐには保存されません）。
- 保存していない変更があるまま別のタブに移ろうとしたり、ウィンドウを閉じようとすると、確認が出ます。
- 「記録の保存先」は、来場者の画面・管理ソフトなど**すべての記録をまとめて入れておくフォルダ**を指定します。
