# 当日運用（刀根山パソコン部）

展示当日の会場・体制・運営の流れをまとめます。
（このサイトは公開されているため、個人名・連絡先は載せず、役割で書いています。随時更新します。）

## 会場と機材

- **生徒用 PC: 40 台**
- **モニター**: 各 PC にフル HD（1920×1080）が 1 枚
- **コントローラー**: 数台を共用。来場者がスタッフに声をかけると、スタッフがその席の PC にコントローラーを挿します
  （アプリから直接スタッフを呼ぶ機能は準備中 → [#83](https://github.com/ken1208git/TonePrism/issues/83)）。
- **データの置き場**: 学校のファイルサーバーに **TonePrism フォルダを丸ごと**置き、各 PC はサーバー上のものを使います
  （ゲーム・登録データ・バックアップをサーバーで一元管理）。

!!! warning "管理ソフトは同時に 1 台だけ"
    データをサーバーで共有しているため、**管理ソフト（Manager）を複数の PC から同時に開かない**でください
    （データ破損のおそれ）。詳しくは [日々の起動と終了](../usage/startup.md#管理ソフトは-1-台のパソコンで-1-つだけ) を参照。

## 当日の体制・役割

交代・シフト制で回します。役割は次のとおりです。

| 役割 | 仕事 |
| --- | --- |
| 呼び込み | 入口で来場を案内・誘導する |
| 来客配置 | 空いている席へ案内する |
| 対応 | 操作の案内、コントローラーの差し込み、トラブル対応 |
| 監視 | 各 PC の状態をまとめて見守る（先生 PC 向け監視ソフト **Monitor の実装後**に運用。現在は準備中） |

## 開場前の準備（チェックリスト）

- [ ] 各 PC から学校のファイルサーバーが見える（アクセスできる）か確認する
- [ ] 各 PC で、サーバー上の **`Launcher.bat`** を立ち上げる
- [ ] 画面が正しく表示されるか（フル HD で崩れていないか）確認する
- [ ] 音が出るか確認する
- [ ] 共用コントローラー（数台）の動作を確認する
- [ ] ゲームがいくつか起動できるか確認する
- [ ] スタッフの役割分担・シフトを確認する

!!! note "準備中: 開場前チェックのワンクリック化"
    今後、スタッフ用の点検画面（サービスモード）に、**この開場前チェックをワンクリックでまとめて実行する機能**を追加する予定です
    （サービスマン動線の強化 → [#74](https://github.com/ken1208git/TonePrism/issues/74)）。実装されたら手順をここに追記します。

## 来場者への対応

- 来場者がゲームを選んで遊べるよう案内します。操作に困っていそうなら声をかけます。
- コントローラーで遊びたい来場者がいたら、スタッフが共用コントローラーをその席の PC に挿します。
- ゲームが固まった・PC の調子がおかしいときは [困ったときは](../usage/troubleshooting.md) を参照して対応します。

（混雑時の待ち列・時間制限などのローカルルールは、決まり次第ここに追記予定です。）

## 閉場・片付け

- 各 PC の Launcher を閉じます。
- 共用コントローラーを回収します。
- データはサーバー上にあるので、各 PC からの持ち帰りは不要です。
- 必要に応じて、管理ソフトの [バックアップ](../usage/manager.md#バックアップ) で控えが取れているか確認します。
